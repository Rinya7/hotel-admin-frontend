# Layouts (Макети сторінок)

## Призначення папки

Папка містить layout компоненти - обгортки для сторінок, які визначають загальну структуру (header, sidebar, navigation) для різних ролей користувачів.

## Структура

```
src/layouts/
├── AdminLayout.vue        # Layout для admin/editor
└── SuperAdminLayout.vue   # Layout для superadmin
```

## Файли

### AdminLayout.vue

**Призначення:** Макет сторінок для admin та editor користувачів.

**Що містить:**

- **Header:**
  - Логотип готелю
  - Назва готелю
  - Перемикач мови
  - Інформація про користувача
  - Кнопка виходу

- **Navigation:**
  - Dashboard
  - Кімнати
  - Аудит центр

- **Content:**
  - `<RouterView />` - відображення поточної сторінки

- **Footer:**
  - Додаткова інформація (опціонально)

**Використання:**

```typescript
// В router/index.ts
{
  path: "/",
  component: AdminLayout,
  meta: { roles: ["admin", "editor"] },
  children: [
    // Дочірні маршрути
  ],
}
```

**Функції:**

- Автоматичне перенаправлення при неавторизованому доступі
- Перевірка ролі користувача
- Відображення навігації залежно від ролі

### SuperAdminLayout.vue

**Призначення:** Макет сторінок для superadmin користувача.

**Що містить:**

- **Header:**
  - Логотип системи
  - Назва системи
  - Перемикач мови
  - Інформація про користувача
  - Кнопка виходу

- **Navigation:**
  - Dashboard
  - Готелі

- **Content:**
  - `<RouterView />` - відображення поточної сторінки

**Використання:**

```typescript
// В router/index.ts
{
  path: "/sa",
  component: SuperAdminLayout,
  meta: { roles: ["superadmin"] },
  children: [
    // Дочірні маршрути
  ],
}
```

**Функції:**

- Доступ тільки для superadmin
- Спеціальна навігація для управління готелями

## Принципи роботи

- **Один layout на роль** - кожна роль має свій layout
- **RouterView** - layout містить `<RouterView />` для відображення дочірніх маршрутів
- **Мета-дані** - layout використовує meta з роутера для перевірки доступу
- **Загальна структура** - header, navigation, content визначені в layout

## Створення нового layout

1. Створити файл з ім'ям Layout
2. Додати структуру (header, navigation, content)
3. Використати `<RouterView />` для дочірніх маршрутів
4. Додати перевірку доступу
5. Підключити в router з meta.roles
