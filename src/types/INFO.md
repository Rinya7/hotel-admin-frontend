# Types (TypeScript типи)

## Призначення папки

Папка містить TypeScript типи та інтерфейси для типізації даних в додатку. Типи забезпечують типобезпеку та покращують читабельність коду.

## Структура

```
src/types/
├── auth.ts          # Типи для авторизації
├── common.ts        # Загальні типи
├── dto.ts           # DTO типи (запити/відповіді)
├── entities.ts      # Типи сутностей (Room, Stay, тощо)
├── hotel.ts         # Типи для готелів
├── rooms.ts         # Типи для кімнат
└── stays.ts         # Типи для проживань
```

## Файли

### auth.ts

**Призначення:** Типи для авторизації та користувачів.

**Що містить:**

- `UserRole` - тип ролі ("superadmin" | "admin" | "editor")
- `LoginRequest` - запит на логін
- `LoginEnvelope` - обгортка відповіді логіну
- `LoginPayload` - дані користувача після логіну

**Використання:**

```typescript
import type { UserRole, LoginRequest } from "@/types/auth";
```

### common.ts

**Призначення:** Загальні типи, які використовуються по всьому додатку.

**Що містить:**

- Загальні інтерфейси
- Утилітарні типи
- Типи для спільних структур

### dto.ts

**Призначення:** Типи для Data Transfer Objects - структури даних для API запитів та відповідей.

**Що містить:**

- Типи для запитів (CreateRoomRequest, CreateStayRequest, тощо)
- Типи для відповідей (RoomResponse, StayResponse, тощо)
- Відповідають backend DTO

### entities.ts

**Призначення:** Типи для сутностей бази даних (Room, Stay, Admin, тощо).

**Що містить:**

- `Room` - тип кімнати
- `Stay` - тип проживання
- `Admin` - тип користувача
- Відповідають backend entities

### hotel.ts

**Призначення:** Типи для роботи з готелями.

**Що містить:**

- `PublicAdminUser` - публічна інформація про готель
- Типи для створення/редагування готелю
- Типи для superadmin операцій

### rooms.ts

**Призначення:** Типи для роботи з кімнатами.

**Що містить:**

- `RoomStatus` - статуси кімнат
- Типи для створення/редагування кімнат
- Типи для операцій з кімнатами

### stays.ts

**Призначення:** Типи для роботи з проживаннями.

**Що містить:**

- `StayStatus` - статуси проживань
- `StayGuest` - інформація про гостя
- Типи для створення/редагування проживань
- Типи для операцій з проживаннями

## Принципи

- **Відповідність backend** - типи відповідають backend типам
- **Строга типізація** - немає `any` типів
- **Перевикористання** - типи використовуються в багатьох місцях
- **Документація** - складні типи мають коментарі

## Створення нового типу

1. Визначити в якому файлі має бути тип
2. Створити інтерфейс або type
3. Додати коментарі для складних типів
4. Експортувати тип
5. Використати в компонентах, stores, API
