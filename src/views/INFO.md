# Views (Сторінки додатку)

## Призначення папки

Папка містить Vue компоненти сторінок (views) - основні екрани додатку, які відображаються через роутер. Кожна view відповідає за конкретну сторінку додатку.

## Структура

```
src/views/
├── audit/              # Аудит центр
│   └── AuditCenter.vue
├── sa/                 # Superadmin сторінки
│   ├── SaDashboard.vue
│   ├── SaHotelsList.vue
│   └── SaHotelDetail.vue
├── Dashboard.vue      # Dashboard для admin/editor
├── Login.vue           # Сторінка логіну
├── RoomsList.vue       # Список кімнат
├── RoomStays.vue       # Проживання по кімнаті
└── StayCheckIn.vue     # Форма заселення
```

## Файли

### Login.vue

**Призначення:** Сторінка авторизації користувача.

**Що робить:**

- Форма логіну (username, password)
- Валідація полів
- Відправка запиту на логін
- Обробка помилок
- Перенаправлення після успішного логіну

**Маршрут:** `/login` (публічний)

### Dashboard.vue

**Призначення:** Головна сторінка для admin/editor зі статистикою та поточними проживаннями.

**Що показує:**

- Статистичні картки (вільні кімнати, зайняті, на прибиранні)
- Сьогоднішні заїзди та виїзди
- Об'єднана таблиця всіх проживань
- Фільтрація по типах (Current, Arrivals, Departures)
- Кольорове кодування типів

**Маршрут:** `/` (для admin/editor)

### RoomsList.vue

**Призначення:** Список всіх кімнат готелю з можливістю управління.

**Що показує:**

- Таблиця кімнат з фільтрацією
- Створення нової кімнати
- Редагування кімнати
- Зміна статусу кімнати
- Історія кімнати
- Масові операції (Wi-Fi, політики)

**Маршрут:** `/rooms` (для admin/editor)

### RoomStays.vue

**Призначення:** Список проживань для конкретної кімнати.

**Що показує:**

- Історія проживань кімнати
- Створення нового проживання
- Редагування проживання
- Операції з проживаннями (check-in, check-out, cancel)

**Маршрут:** `/rooms/:roomNumber/stays` (для admin/editor)

### StayCheckIn.vue

**Призначення:** Форма заселення з детальною інформацією про гостей.

**Що показує:**

- Інформація про проживання
- Форма для додавання гостей
- Детальна інформація про кожного гостя (документи, дата народження)
- Збереження та check-in

**Маршрут:** `/rooms/:roomNumber/stays/:id/checkin` (для admin/editor)

### audit/AuditCenter.vue

**Призначення:** Центр аудиту з логами всіх змін в системі.

**Що показує:**

- Таблиця логів змін
- Фільтрація по типу (room, stay)
- Фільтрація по користувачу, ролі, даті
- Детальна інформація про кожну зміну

**Маршрут:** `/audit` (для admin/editor)

### sa/ (Superadmin сторінки)

**SaDashboard.vue**
- Призначення: Dashboard для superadmin
- Маршрут: `/sa`
- Функції: Загальна статистика по всіх готелях

**SaHotelsList.vue**
- Призначення: Список всіх готелів
- Маршрут: `/sa/hotels`
- Функції: Створення нового готелю, перегляд списку

**SaHotelDetail.vue**
- Призначення: Детальна інформація про готель
- Маршрут: `/sa/hotels/:id`
- Функції: Редагування готелю, управління адмінами та редакторами

## Принципи організації

- **Одна view = одна сторінка** - кожна view відповідає за одну сторінку
- **Роутинг** - views підключені через Vue Router
- **Layouts** - views використовують layouts (AdminLayout, SuperAdminLayout)
- **Компоненти** - views використовують компоненти з `src/components/`
- **Stores** - views використовують Pinia stores для стану

## Створення нової view

1. Створити файл в відповідній папці
2. Додати маршрут в `src/router/index.ts`
3. Використати layout (якщо потрібно)
4. Використати компоненти та stores
5. Додати локалізацію
