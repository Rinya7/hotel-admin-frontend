# Components (Vue компоненти)

## Призначення папки

Папка містить перевикористовувані Vue компоненти, організовані за категоріями для зручної навігації та підтримки.

## Структура

```
src/components/
├── common/              # Загальні компоненти
│   ├── AuditLogViewer.vue
│   ├── CreateRoomForm.vue
│   └── SortIcon.vue
├── dashboard/           # Компоненти для dashboard
│   └── StatCard.vue
├── rooms/               # Компоненти для кімнат
│   └── RoomHistory.vue
├── stays/               # Компоненти для проживань
│   └── StayHistory.vue
├── superadmin/          # Компоненти для superadmin
│   ├── CreateHotelForm.vue
│   └── EditHotelDialog.vue
├── ui/                  # UI компоненти
│   ├── Button.vue
│   └── Notification.vue
├── GlobalLoader.vue      # Глобальний loader
├── LanguageSwitcher.vue # Перемикач мови
├── NotificationContainer.vue # Контейнер сповіщень
└── Spinner.vue          # Спиннер завантаження
```

## Категорії компонентів

### common/ (Загальні компоненти)

**AuditLogViewer.vue**
- Призначення: Відображення аудит логів у вигляді таблиці
- Використання: В AuditCenter view
- Функції: Фільтрація, сортування, відображення деталей

**CreateRoomForm.vue**
- Призначення: Форма створення нової кімнати
- Використання: В RoomsList view
- Функції: Валідація, відправка даних, обробка помилок

**SortIcon.vue**
- Призначення: Іконка сортування для таблиць
- Використання: В таблицях для індикації напрямку сортування
- Функції: Відображення напрямку (asc/desc/none)

### dashboard/ (Dashboard компоненти)

**StatCard.vue**
- Призначення: Картка зі статистикою
- Використання: В Dashboard view
- Функції: Відображення числа, заголовка, іконки, кольору

### rooms/ (Компоненти кімнат)

**RoomHistory.vue**
- Призначення: Історія змін статусів кімнати
- Використання: В RoomsList view
- Функції: Відображення логів, фільтрація, деталі

### stays/ (Компоненти проживань)

**StayHistory.vue**
- Призначення: Історія змін статусів проживання
- Використання: В RoomStays view
- Функції: Відображення логів, фільтрація, деталі

### superadmin/ (Superadmin компоненти)

**CreateHotelForm.vue**
- Призначення: Форма створення нового готелю (адміна)
- Використання: В SaHotelsList view
- Функції: Валідація, геокодування адреси, Wi-Fi поля, політики

**EditHotelDialog.vue**
- Призначення: Діалог редагування готелю
- Використання: В SaHotelDetail view
- Функції: Редагування всіх полів готелю, валідація

### ui/ (UI компоненти)

**Button.vue**
- Призначення: Перевикористовувана кнопка
- Використання: По всьому додатку
- Функції: Різні варіанти (primary, secondary, danger), disabled стан

**Notification.vue**
- Призначення: Компонент сповіщення
- Використання: В NotificationContainer
- Функції: 4 типи (success, error, warning, info), анімації, авто-закриття

### Глобальні компоненти

**GlobalLoader.vue**
- Призначення: Глобальний індикатор завантаження
- Використання: В App.vue
- Функції: Показується під час HTTP запитів

**LanguageSwitcher.vue**
- Призначення: Перемикач мови інтерфейсу
- Використання: В layouts
- Функції: Перемикання між uk, en, it, de

**NotificationContainer.vue**
- Призначення: Контейнер для відображення сповіщень
- Використання: В App.vue
- Функції: Управління позиціонуванням, відображенням сповіщень

**Spinner.vue**
- Призначення: Спиннер завантаження
- Використання: В компонентах для локального завантаження
- Функції: Анімація завантаження

## Принципи організації

- **Категорії** - компоненти організовані за функціональністю
- **Перевикористання** - компоненти можна використовувати в різних місцях
- **Ізоляція** - кожен компонент має свою відповідальність
- **Типізація** - всі props типізовані через TypeScript

## Створення нового компонента

1. Визначити категорію (common, ui, rooms, тощо)
2. Створити файл з ім'ям компонента
3. Додати TypeScript типи для props
4. Додати локалізацію (якщо потрібно)
5. Використати в views або інших компонентах
